DeployX 是一个面向内部团队的轻量级 DevOps 应用交付平台，主要服务于研发、测试和运维人员。
它围绕“应用”这一核心资产，提供从应用创建、部署配置、容器启动、状态监控、访问跳转、更新发布到部署记录追踪的一体化能力。

## 为什么需要 DeployX

在日常交付过程中，开发人员通常需要手动维护 Dockerfile、docker-compose.yml 或 Kubernetes YAML，还要处理配置文件、环境变量、中间件依赖和目标服务器信息。

这些工作重复、容易出错，也会让应用交付依赖少数熟悉部署细节的人。

DeployX 希望把这些流程收敛到统一的平台中，通过可视化表单和部署向导，让团队可以用更标准、更可追踪的方式完成应用发布。



## 核心能力

- 应用创建与基础信息管理
- 部署配置和运行参数维护
- 容器启动、停止和状态查看
- 应用访问地址跳转
- 应用更新发布
- 部署记录追踪
- 多环境配置管理
- 后续可扩展 Docker、Docker Compose 和 Kubernetes 部署资源生成


## 项目组成

当前 DeployX 仓库包含三个主要项目：

- Backend：Go 后端服务，负责配置、接口、部署任务和平台能力编排
- Frontend：React + Vite 前端项目，负责应用管理和部署操作界面
- Docs：Next.js + Fumadocs 文档站，负责项目说明、使用指南和开发文档
