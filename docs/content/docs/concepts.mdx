# 核心概念

DeployX 围绕应用交付流程组织平台能力。理解这些核心概念，可以帮助你更快看懂后续的使用指南和开发文档。

## 应用

应用是 DeployX 管理的核心资产，代表一个需要被构建、部署、运行和维护的服务或系统。

## 环境

环境用于区分不同运行阶段，例如开发环境、测试环境和生产环境。不同环境可以拥有不同的配置、依赖和部署目标。

## 部署

部署是将应用发布到指定环境的过程。它包含部署配置、运行参数、目标服务器或集群信息，以及执行状态。

## 发布记录

发布记录用于追踪每一次部署动作，包括发布时间、部署人、目标环境、版本信息和执行结果。
